## vs code 编辑markdown的三个插件使用方法
#### 1. Markdown All in One（核心编辑工具）
核心作用：提供快捷键、目录生成、格式自动补全，提升 Markdown 编写效率（1100万+安装量的核心原因）。

##### 🔧 核心功能+使用方法
| 功能          | 操作步骤/快捷键                                                                 | 效果说明                                                                 |
|---------------|--------------------------------------------------------------------------------|--------------------------------------------------------------------------|
| 常用格式快捷键 | 加粗：`Ctrl+B`/`Cmd+B` <br> 斜体：`Ctrl+I`/`Cmd+I` <br> 代码块：`Ctrl+Shift+K`  | 选中文字后按快捷键，直接生成对应格式（如 `**加粗**`、`*斜体*`、```代码块```） |
| 列表快速生成  | 输入 `-`/`1.` 后按空格 → 自动生成无序列表/有序列表； <br> 按 `Tab` 缩进子列表，`Shift+Tab` 取消缩进 | 无需手动写格式，快速搭建列表结构（支持多级列表）                           |
| 链接/图片插入 | 链接：`Ctrl+K` → 输入 `[显示文本](链接地址)` <br> 图片：`Ctrl+Shift+I` → 输入 `![alt文本](图片路径)` | 自动补全链接/图片语法，直接填空即可（配合 PicGo 可自动上传 GitHub 图床）  |
| 目录（TOC）生成 | 在文档任意位置输入 `[TOC]` → 按 `Enter` 或保存文件 | 自动生成基于标题（#~######）的目录，点击目录可跳转对应章节                 |
| 目录更新      | 目录生成后，标题修改/新增时，按 `Ctrl+Shift+P`（命令面板）→ 搜索「Markdown All in One: Update TOC」 | 一键同步目录与标题，避免手动修改                                         |
| 标题层级调整  | 选中标题文字 → 按 `Ctrl+Shift+[`（降低层级，如 #→##）/ `Ctrl+Shift+]`（提升层级） | 快速调整标题级别，无需手动修改 `#` 数量                                   |
| 公式块插入    | 按 `Ctrl+Shift+M` → 生成 `$$ 公式 $$`（支持 LaTeX 语法）                        | 适合编写技术文档中的数学公式，与 GitHub 支持的公式渲染兼容                 |

##### ✨ 进阶技巧
- 自定义快捷键：按 `Ctrl+K Ctrl+S` 打开快捷键设置，搜索「Markdown All in One」，可修改任意功能的快捷键（如把「更新目录」改为 `Ctrl+U`）；
- 启用自动补全：默认已开启，输入 `#`/`-`/`[` 等符号时，会自动提示后续语法（如输入 `[` 后提示 `[文本](链接)`）；
- 导出 PDF/HTML：按 `Ctrl+Shift+P` → 搜索「Markdown All in One: Print current document to PDF」，可直接导出文档（需系统安装打印机驱动）。


#### 2. Markdown Preview Github Styling（GitHub 风格预览）
核心作用：让 VS Code 内置的 Markdown 预览效果，和 GitHub 上的渲染效果完全一致（避免「本地预览好看，GitHub 显示错乱」）。

##### 🔧 核心功能+使用方法
| 功能          | 操作步骤                                                                 | 效果说明                                                                 |
|---------------|--------------------------------------------------------------------------|--------------------------------------------------------------------------|
| 打开 GitHub 风格预览 | 方法1：按 `Ctrl+Shift+V`（默认预览快捷键），自动使用 GitHub 样式； <br> 方法2：点击文档右上角的「预览」图标（眼睛图标） | 预览窗口会显示和 GitHub 完全一致的样式（字体、间距、列表样式、代码高亮等） |
| 分屏编辑+预览 | 打开预览后，拖动预览窗口到右侧/下方，实现「左边编写，右边实时预览」         | 实时同步编辑内容，预览窗口会自动刷新（延迟≤1秒）                           |
| 验证 GitHub 兼容性 | 编写时重点关注：表格、代码块、数学公式、任务列表 → 预览窗口直接看效果       | 例如：GitHub 支持的「合并单元格表格」「语法高亮代码块」，预览中会精准还原 |

##### ✨ 进阶技巧
- 强制使用 GitHub 样式：如果预览仍显示默认样式，按 `Ctrl+Shift+P` → 搜索「Markdown: Change preview style」→ 选择「GitHub」（插件安装后默认已设置）；
- 调整预览字体大小：预览窗口中按 `Ctrl++`/`Ctrl+-` 放大/缩小，适配你的屏幕；
- 支持 GitHub 特有样式：如 GitHub 支持的「任务列表」（`- [x] 完成`/`- [ ] 未完成`）、「引用块嵌套」，预览中会完全还原 GitHub 的显示效果。


#### 3. GitHub Markdown（GitHub 语法适配+增强）
核心作用：补充 GitHub 特有的 Markdown 语法支持（VS Code 原生语法可能不完整），确保编写的文档在 GitHub 上 100% 兼容。

##### 🔧 核心功能+使用方法
| 功能          | 操作步骤+示例                                                             | 效果说明                                                                 |
|---------------|--------------------------------------------------------------------------|--------------------------------------------------------------------------|
| 支持 GitHub 特有语法 | 直接编写以下语法，插件会自动识别并渲染： <br> 1. 任务列表：`- [x] 完成文档`/`- [ ] 待优化` <br> 2. 表格合并：`| 表头1 | 表头2 |`+`| --- | --- |`（GitHub 支持的标准表格） <br> 3. 代码块语法高亮：```python（指定语言）+ 代码 + ``` <br> 4. 数学公式：`$E=mc^2$`（行内公式）/ `$$\sum_{i=1}^n i$$`（块级公式） | 编写时无需担心语法不兼容，GitHub 支持的语法插件都能识别，预览中精准还原 |
| 跳转链接优化 | 编写仓库内文件链接：`[跳转到其他文档](./other.md)` → 按住 `Ctrl` 点击链接，可直接在 VS Code 中打开目标文件 | 方便管理多文件的 GitHub 仓库（如项目文档、Wiki 等）                       |
| 图片路径兼容 | 支持 GitHub 图床路径：`![alt](https://raw.githubusercontent.com/用户名/仓库名/分支/图片路径.png)` | 配合 PicGo 上传图片后，直接粘贴路径，插件确保渲染正常                     |

##### ✨ 进阶技巧
- 语法校验：编写时如果语法不符合 GitHub 规范（如表格格式错误），插件会在编辑器中提示警告（红色波浪线），鼠标悬停可查看错误原因；
- 自动补全 GitHub 语法：输入 `- [ ` 时，自动提示 `x`/` `（任务列表）；输入 ``` 时，自动提示常用编程语言（如 python、javascript），快速生成语法高亮代码块；
- 适配 GitHub Pages：如果你的文档要部署到 GitHub Pages，编写时使用插件支持的语法（如目录 `[TOC]`、公式、表格），部署后会和本地预览完全一致。


### 三、三个插件配合使用流程（核心 workflow）
结合「编写→预览→提交 GitHub」的完整场景，推荐流程：
1. **编写文档**：用 `Markdown All in One` 的快捷键（如 `Ctrl+B` 加粗、`Ctrl+Shift+I` 插图片）快速编写，用 `[TOC]` 生成目录；
2. **预览效果**：按 `Ctrl+Shift+V` 打开预览，通过 `Markdown Preview Github Styling` 确认样式和 GitHub 一致，检查表格、代码块是否正常；
3. **语法兼容**：用 `GitHub Markdown` 补全 GitHub 特有语法（如任务列表、公式），确保无语法警告；
4. **提交 GitHub**：通过 VS Code 内置 Git（左侧边栏「源代码管理」）提交文档，推送到 GitHub 仓库，最终显示效果和本地预览完全一致。


### 四、常见问题排查
1. 预览不是 GitHub 风格？
   - 解决方案：按 `Ctrl+Shift+P` → 搜索「Markdown: Change preview style」→ 选择「GitHub」；若仍无效，重启 VS Code 即可。
2. 目录不更新？
   - 解决方案：按 `Ctrl+Shift+P` → 搜索「Markdown All in One: Update TOC」，或重新保存文件（部分情况下自动更新）。
3. 快捷键冲突？
   - 解决方案：按 `Ctrl+K Ctrl+S` 打开快捷键设置，搜索冲突的快捷键（如 `Ctrl+B`），修改 `Markdown All in One` 的对应快捷键即可。
4. GitHub 上图片不显示？
   - 解决方案：确保图片路径是 GitHub 可访问的（如 GitHub 图床路径、仓库内相对路径），避免使用本地绝对路径（如 `C:\xxx\img.png`）。


通过这三个插件的配合，你可以在 VS Code 中实现「高效编写+GitHub 无缝适配」，无需切换工具就能完成从文档创作到 GitHub 发布的全流程，非常适合开发者编写技术文档、博客或开源项目说明！
