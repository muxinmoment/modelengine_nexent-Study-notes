华为云modelengine具体安装步骤：
# 可以去看model engine的下载官网教程
1.华为云上多了一个把下载到本地的安装包和下载脚本上传到华为云目录的工作

2.创建软连接python指向python3
# 1. 确认 python3 路径
which python3          # 示例输出 /usr/bin/python3

# 2. 创建软链（update-alternatives 方式，最干净）
sudo update-alternatives --install /usr/bin/python python /usr/bin/python3 1

# 3. 验证
python --version       # 应输出 Python 3.x.x

3.进入软件包目录，例如/mnt/ModelEngine
cd /mnt/ModelEngine
chmod a+x install_modelengine*.sh

4.根据环境不同，选择以下一条命令执行，根据参数说明替换参数
# 非 A3000 环境
bash install_modelengine*.sh --ns=<namespace> --workdir=<workdir> --storage-path=<storage-path> --ip=<ip> --port=<port> --password=<password>

# A3000 环境
bash install_modelengine*.sh --ns=<namespace> --workdir=<workdir> --storage-path=<storage-path> --ip=<ip> --port=<port> --password=<password> --a3000

# 我的是：
bash install_modelengine*.sh --ns=model-engine --workdir=/mnt/ModelEngine --storage-path=/mnt/model-engine --ip=192.168.0.204 --port=30000 --password=@Zenga321

4.tips:

需要手动输入前端登录密码
新开启一个窗口，并执行以下命令，实时查看运行状态 watch -n1 kubectl get pod -n <namespace>

# 我的是：
watch -n1 kubectl get pod -n model-engine

# 现在遇到了一个问题，使用这个命令之后发现第一个容器可以运行，但是到第二个就报错
检查了日志之后，原因如下：
脚本没“写错”，只是它以为容器里能随便“新建文件夹、改文件、切换用户”，而 Kubernetes 把容器设成了“只读 + 非 root”，结果第二步的 gaussdb 容器啥也干不了，就疯狂报错。
把房子比做容器：
脚本想在屋里刷墙、打洞、换锁（mkdir、chmod、su）。
你偏偏给房子贴了封条：地板不能钻（只读），而且不是房东本人（非 root）在场，当然啥都干不成。
于是工人（gaussdb 启动脚本）在门口喊“Permission denied！”——就是你现在看到的刷屏。

问题还是这个问题，经过我的仔细研究，这个问题是华为安装脚本的问题，直接用他们开发环境的权限，但是这个在标准部署的k8s上面并不可行
启动脚本通用逻辑里认为：
"任何我要写的配置目录，都必须 700 权限 + 属主必须是 gaussdb 用户"
于是上来就 chmod 700 /opt/huawei/fce/runtime/security/priv/gaussdb.conf
chown gaussdb:gaussdb /opt/huawei/fce/runtime/security/priv/gaussdb.conf
早期版本（或开发环境）这个目录就是 hostPath/emptyDir，可写可改，脚本跑得很开心；
到了生产/当前版本，为了 统一证书管理、方便滚动升级，平台把同目录改成了 ConfigMap/Secret 挂载 → 内核强制只读，但脚本没做「只读场景」的兼容，于是照跑不误，直接翻车。
